# DBOBJ 設計変更仕様書
Date: 2026-06-10

## 概要

[design-concept.md](../../design/design-concept.md)（2026-06-10 改訂）の To Do を具体化する。

- TableTools を廃止し、メタ付き AoH の扱いを MetaAoh（`lib/MetaAoh.pm`）に置き換える
- `hashes()` は metaAoh（MetaAoh オブジェクト）を返す
- `spool($spool_id)` を廃止する

## 変更の背景

- TableTools は廃止が決まり、後継として MetaAoh（meta をオブジェクト自身に持つ AOH）が導入された
- 従来の「先頭にメタ行 `{'#' => {...}}` を置いた素の配列」形式は TableTools の規約であり、TableTools とともに廃止する
- スプールへの書き出しは DBOBJ の責務ではない。呼び出し側が `hashes()` の metaAoh を使って外で行う
- DBOBJ の本質は「SQL の結果（テーブル構造）を metaAoh として吐き出すこと」である

## API 変更

### 廃止

| メソッド | 理由 |
|---|---|
| `spool($spool_id)` | スプールへの書き出しは DBOBJ の責務外。呼び出し側が metaAoh を使って外で行う |

### 変更

| メソッド | 旧 | 新 |
|---|---|---|
| `hashes()` | `[]` または `[{'#' => {attrs, order, count}}, {row}, ...]` | `metaAoh`（0件でも空の metaAoh を返す） |

### 変更なし

`new`, `prepare`, `execute`, `run`, `get`, `list`, `arrays`, `psql`, `close`

## 設計方針

### MetaAoh 連携

- `hashes()` はステートメントハンドルから AOH を取得し、`MetaAoh->new($aoh, @order)` で metaAoh を生成して返す
- `@order` のカラム指定は DBI の型情報から組み立てる。`'num'`（REAL・INTEGER・NUMERIC 系の型コード）のカラムは `NAME#`、`'str'` のカラムは `NAME` とする
- カラムの並びはステートメントハンドルの `NAME` の順序に従う
- 返る metaAoh の meta は `order`・`cols`・`attrs`・`grouped` を持つ（MetaAoh の仕様による）。件数は `count()` メソッドで得る

### 0件時の挙動

- `hashes()` は 0件でも空の metaAoh を返す。「0件なら `[]` を返し meta も持たない」という旧仕様は廃止する
- 0件でもステートメントハンドルからカラム名・型は取得できるため、meta（`order`・`cols`・`attrs`）は保持され、`count()` は 0 を返す

### NULL の扱い

- 従来どおり、取得値の `undef` はすべて `''` に変換してから返す
- これは MetaAoh の AOH 条件「値に `undef` を含まない」とも整合する

### グループ化

- 呼び出し側が metaAoh の `group()` を使う。DBOBJ は関与しない

### 依存関係

- `use Spool` を削除し、`use MetaAoh` を追加する
- TableTools は廃止のため依存ゼロとする。Spool への依存もなくす
- CommonIO はすべての基盤となるライブラリであり、積極的に使用する（仕様は `lib/CommonIO.spec.md` に従う）。`use CommonIO` を追加する
- 不要になる `lib/TableTools.pm`・`lib/Spool.pm` の除去はユーザーが行う（lib 管理）

### エラー処理（CommonIO の活用）

- エラーは従来どおり手動検知とし、素の `die` を CommonIO の `dying()` に置き換える。`dying()` は error レベルのログをファイルに残してからトレースバック付きで例外を投げる
- ログ出力先は環境変数 `LOGDIR`（定義元は `.claude/settings.json` の `env`）に従う
- DBOBJ の責務にファイル入出力・fork はないため、CommonIO のその他の API（`out_file`・`write_do`・`run_in_fork` 等）は現時点で使用箇所なし

### 実装上の変更点

- `spool()` メソッドと、spool 専用の処理を削除する
- meta 組み立て（旧 `_build_meta`）は MetaAoh のカラム指定の組み立てに置き換える
- すべての `die` を CommonIO の `dying()` に置き換える
- `src/DBOBJ.pm` 冒頭の Terms / Rules コメントを design-concept.md の `Rules` と同期する
- `test/dbobj.t` の spool 系テスト（旧 spec#22〜24）を削除し、hashes 系テストを metaAoh 検証に書き換える
- `test/insert.sql`・`test/error.sql`・`test/notice.sql` は psql 用のため残す
- README.md・README_ja.md・docs/spec.md・docs/test-spec.md を本変更に合わせて更新する

## テスト方針

- 実 PostgreSQL DB（`develop`）を使う結合テスト（モックなし）
- 環境変数（`PGHOST`, `PGPORT`, `PGUSER`, `PGPASSWORD`）が設定済みであること（定義元は `.claude/settings.json` の `env`）
